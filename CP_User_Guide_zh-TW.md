# CP Team 使用手冊：ST26 論壇 HTML 產生器

本工具用於協助 CP Team 製作 SEMICON Taiwan 論壇頁面的中英文 HTML。使用者不需要接觸程式碼，只要貼上資料、補齊內容與 Logo，即可產出可貼入 Drupal 的 HTML。

線上工具：

https://semitwmkt.github.io/SEMI-AutoCompiler-FORUM/

## 使用前準備

請先準備：

- 論壇 Excel 資料，通常來自 `ST26_官網連結` 資料表
- 論壇中文與英文名稱
- 論壇主題與簡介
- 場地資訊
- 報名連結
- 主辦、共同主辦、贊助單位 Logo
- 論壇主席與主持人資料，如有

## 基本流程

1. 開啟產生器
2. 貼上 Excel 資料
3. 點選「解析資料」
4. 選擇要製作的論壇場次
5. 進入「確認 & 編輯」
6. 補齊文字、連結、人員與 Logo
7. 點選「產生 HTML」
8. 檢查預覽
9. 複製中文版或英文版 HTML
10. 貼入 Drupal 對應頁面

## Step 1：輸入資料

### 使用 Excel 模式

適用於已有標準資料表時。

1. 從 Excel 複製 `ST26_官網連結` 的資料
2. 貼到「貼上資料」欄位
3. 點選「解析資料」
4. 從下拉選單選擇正確論壇
5. 確認場地與票價預覽
6. 點選「繼續」

### 使用文字 / AI 模式

適用於只有 email、Word 段落或非標準文字時。

注意：

- 需要自行輸入 API Key
- API Key 不會被寫入 GitHub
- 請勿共用個人 API Key 給公開使用者
- AI 解析後仍須人工確認

## Step 2：確認與編輯

請依序檢查：

- 場次英文名稱
- 場次中文名稱
- 場地英文
- 場地中文
- 中文主題
- 英文主題
- 中文大綱 / 簡介
- 英文大綱 / 簡介
- 指導單位
- 論壇主席
- 論壇主持人
- 報名連結
- 地點導航連結
- 備註文字
- Logo 區域

若資料不完整，可以先儲存草稿，等資料到齊後再回來補。

## 草稿功能

草稿適合用於：

- 廠商 Logo 尚未提供
- 主題或簡介還在確認
- 人員資料還未定稿
- 只想先完成部分內容

操作方式：

1. 在「確認 & 編輯」頁面點選「儲存草稿」
2. 下次使用同一台電腦、同一個瀏覽器、同一個網址開啟工具
3. 選回同一個論壇場次
4. 若偵測到草稿，點選「載入上次進度」

注意：

- 草稿存在瀏覽器內，不是在 GitHub 或 Drupal
- 換電腦、換瀏覽器、清除瀏覽器資料後，草稿可能無法保留
- 若 GitHub Pages 網址改變，原本草稿可能讀不到
- 重要內容建議另存於工作文件中備份

## Logo 區域使用方式

Logo 可能來自不同廠商，尺寸、比例、白邊都不一致。產生器提供 Logo 管理功能，協助在網頁上呈現得較一致。

### 新增 Logo

1. 在對應區域點選「新增 Logo」
2. 點擊 Logo 卡片上傳圖片，或貼入圖片 URL
3. 使用大小拉桿調整視覺比例
4. 如 Logo 特別寬，可勾選「聯名 Logo」
5. 使用上下按鈕調整順序

### Logo 區域分類

- 主辦單位 Organizer
- 共同主辦單位 Co-organizer
- 贊助單位 Sponsored by

請確認 Logo 放在正確分類中。

## Logo 切邊與品牌注意事項

目前工具會協助處理 Logo 周圍的空白邊，目的是讓不同廠商提供的圖片在網頁上看起來更整齊。

但請注意：

- Logo 是品牌資產，若廠商或品牌規範要求保留固定留白，應以品牌規範為準
- 若不確定能否裁切白邊，建議請廠商提供「適合網頁露出的透明背景版本」
- 不建議裁切到 Logo 本體、文字、圖形或註冊商標符號
- 若 Logo 周圍留白是刻意設計的一部分，請保留原始檔案或調整大小，不要強行改變比例
- 產出前請用預覽檢查 Logo 是否完整、清楚且未變形

建議原則：

- 可以調整顯示大小
- 可以讓不同 Logo 在同一區域中視覺一致
- 不應改變 Logo 圖形本身
- 不應把品牌規範要求的安全留白當作一般白邊刪除

## Step 3：輸出與預覽

點選「產生 HTML」後，請檢查：

- 中文版是否正確
- 英文版是否正確
- 電腦版預覽是否正常
- 手機版預覽是否正常
- 報名按鈕連結是否正確
- 地點連結是否正確
- Logo 是否完整、比例是否可接受
- 票價資訊是否正確
- 備註與免責聲明是否符合需求

確認後：

1. 點選「複製中文版代碼」
2. 貼入 Drupal 中文頁面
3. 點選「複製英文版代碼」
4. 貼入 Drupal 英文頁面

## 發布前檢查清單

發布前建議逐項確認：

- 中文名稱正確
- 英文名稱正確
- 中文主題正確
- 英文主題正確
- 中英文簡介無錯字
- 場地資訊正確
- Google Maps 連結正確
- 報名連結正確
- 主辦 / 共同主辦 / 贊助 Logo 分類正確
- Logo 沒有被裁到內容
- Logo 順序正確
- 論壇主席與主持人資料正確
- 票價正確
- 手機版預覽沒有跑版
- Drupal 貼上後前台顯示正常

## 常見問題

### Q1：我只要新增一個 Logo，需要重做整個論壇嗎？

不用。若之前有儲存草稿，可以載入草稿後新增 Logo，再重新產生 HTML。

### Q2：為什麼換電腦後找不到草稿？

草稿存在原本電腦的瀏覽器中。換電腦後通常讀不到。重要內容請另外備份。

### Q3：Logo 看起來太大或太小怎麼辦？

使用 Logo 卡片中的大小拉桿調整。若 Logo 是長條形，可勾選「聯名 Logo」。

### Q4：廠商給的 Logo 有很多白邊怎麼辦？

可以先使用工具處理並預覽。但如果廠商有品牌規範，請以品牌規範為準，必要時請廠商提供網頁用檔案。

### Q5：產出的 HTML 可以再手動改嗎？

不建議。若內容要改，請回到產生器修改後重新產出，這樣比較能保持格式一致。

## 建議工作方式

若資料分批到齊，建議：

1. 先用 Excel 建立論壇基本資料
2. 補上已確認的主題、簡介、人員
3. 儲存草稿
4. 等 Logo 或其他資料到齊後載入草稿
5. 補齊資料
6. 產出 HTML
7. 貼入 Drupal
8. 前台檢查

這樣可以避免每次都從頭輸入。
