# ST26 論壇 HTML 產生器更新日誌

## [v6.9] 2026-05-11 - 柔性降級解析與模塊修復
* **【Bug 修復】人員模塊還原**：修復 v6.8 因過度精簡代碼而遺失的「論壇主持人 (Forum Moderator)」區塊，以及主席與主持人的「個人介紹頁 URL」輸入欄位。
* **【架構重構】柔性降級解析引擎 (Graceful Degradation)**：
    - **廢除絕對阻斷機制**：為應對營運團隊可能使用舊版或格式不齊的 Excel 檔案，解析引擎現在能相容任何排列順序或殘缺欄位（包含舊版與最新 0511(L) 版）。
    - **盡力捕捉與手動容錯**：若系統掃描不到 `Theme`、`中文連結` 或 `英文連結` 欄位，將不再報錯鎖死，而是允許放行。系統會於前台預覽區亮起黃色警告（⚠️），提示操作者在後續步驟「手動填補」遺失的資料，保留極致彈性的營運容錯力。

## [v6.8] 2026-05-11 - 核心權導向：Theme 全功能集成版
* **【架構定案】Theme 功能回歸**：確認 Drupal 後台無獨立主題欄位，將 Excel 中的 `Theme` 獨立抽出，作為 HTML 區塊的副標題與大綱摺疊按鈕。
* **【資料流同步】0511(L) 最終公版對齊**：精準掃描並映射 `Theme`、`中文連結`、`英文連結` 三大核心營運欄位。

## [v6.7] 2026-05-11 - 智慧定位掃描引擎
* **【架構革命】全動態表頭掃描**：移除絕對座標 (Index) 抓取邏輯，改為「認字不認欄」，自動尋找關鍵字定位資料，無視 Excel 欄位的左右移動或增刪。

## [v6.6] 2026-05-11 - 票價彈性備註系統
* **【功能新增】動態票價備註**：新增中英文獨立備註欄位。若無填寫則該區塊徹底隱藏，不影響主票價視覺層級。

## [v6.5] 2026-05-11 - 終極自動化：DOM 原生攔截引擎
* **【架構重構】全域掃描與主動綁定**：徹底移除人工「喚醒標籤」。腳本改為直接掃描 Drupal 原生吐出的 Class (`.session__people` 與 `.session__subsession`)，自動點亮導航列。

## [v5.1 - v6.4 歷史摘要]
* 建立全自動 CMS 區塊偵測腳本、雙語動態分流引擎、RWD 卡片化降維排版、LocalStorage 瀏覽器暫存系統。
