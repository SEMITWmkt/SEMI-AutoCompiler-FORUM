ST26 論壇 HTML 產生器更新日誌

[v6.6] 2026-05-11 - 票價彈性備註系統

【功能新增】動態票價備註：為因應特定論壇（如委員會專屬優惠）的特例狀況，於「報名」步驟新增中英文獨立的備註欄位。

【視覺排版】自動折疊與渲染：備註區塊位於票價大標題之下、票價表格之上。若無填寫內容，該區塊將徹底隱藏，不佔用任何版面空間；若有填寫，則以低干擾的視覺樣式（淺底色、14px字級）呈現，確保不影響主票價的視覺層級。

【系統連動】：草稿記憶系統同步支援備註欄位的儲存與讀取。

[v6.5] 2026-05-11 - 終極自動化：DOM 原生攔截引擎

【架構重構】全域掃描與主動綁定：徹底移除人工「喚醒標籤」。腳本改為直接掃描 Drupal 原生吐出的 Class (.session__people 與 .session__subsession)。只要後台建好資料，前台對應的「講師」與「議程」導航按鈕即刻自動亮起並綁定平滑滾動，達到 100% 零摩擦營運。

[v6.4] 2026-05-11 - CMS 結構化欄位解耦

【邏輯轉換】目標反向呼叫錨點：為解決 Drupal 後台無法自訂 HTML ID 的限制，設計了 .st26-trigger-speakers 等「喚醒標籤」。CP 團隊可隨意將標籤貼在任何區塊，由 HTML 的微型腳本全域掃描並自動連動導覽列。(註：此功能已於 v6.5 被更先進的原生掃描取代)

[v6.3] 2026-05-11 - Excel 資料流精準重構

【資料修正】：深度解析最新版 ST26_官網連結.csv。重新映射解析引擎的 Index，校準票價（起始於 c[8]）與中英文連結（c[18], c[19]）的抓取座標，確保資料自動帶入完全精準。

[v6.2] 2026-05-11 - 營運防呆與動態錨點開關

【UI 強化】：於介面「② 主題/大綱」新增手動開啟「講師陣容」與「詳細議程」的防呆開關，避免在後台尚未建置資料時產出無效的死連結。

[v6.0 & v6.1] 2026-05-11 - 雙語分流與 Excel 掛載

【自動化】Excel 連結自動掛載：支援解析 Excel 中的報名連結。

【架構升級】雙語動態分流引擎：輸出 HTML 時，系統自動判斷語系。中文版強制植入中文連結，英文版強制植入英文連結；若任一缺失，則自動套用另一連結作為防呆備案 (Fallback)。

【功能解鎖】全導覽錨點：開啟「票價與報名」、「講師陣容」、「詳細議程」的快速導航列。

[v5.8 & v5.9] 2026-05-07 - 商業視覺與層級重構

【UI 優化】主題按鈕化：將原本套用在「地點」的深色與背景色移植至「主題」區塊，提升白底網頁的點擊暗示 (Affordance)。

【版面淨化】移除地點模塊：因 Drupal 外層模板已涵蓋地點資訊，產生器不再產出地點 Badge。

【視覺降維】Day Pass 弱化：縮小單日通行證的字級並改為灰底，確保「主票價」維持絕對的視覺統治力。

【功能新增】共同主辦單位：新增 Co-organizer 分類，支援聯名 Logo。

【動態渲染】空區塊抹除：若主辦、共同主辦或贊助沒有上傳任何 Logo，最終 HTML 將徹底移除該標題與間距。

[v5.6 & v5.7] 2026-05-07 - RWD 降維與語系隔離

【前端技術】無表格 (Table-less) 卡片化 RWD：徹底廢除 <table> 標籤，改用 CSS Grid。在手機端（<768px）自動降維為卡片式排列，解決文字擠壓與標籤疊影的破版問題。

【語系隔離】完全動態翻譯：將 Advised by、Forum Chair、各階段早鳥票價、Day Pass 等字眼與語系切換連動，確保中文版 HTML 產出 100% 正式的商業中文用語。

[v5.5 以前]

確立基礎架構、導入 Canvas 前端圖片壓縮技術 (Base64)、建置 LocalStorage 瀏覽器暫存系統。
