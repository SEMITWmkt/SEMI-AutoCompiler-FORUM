ST26 論壇 HTML 產生器更新日誌

[v9.3] 2026-06-02 - Logo 標題間距與扁平化輸出微調

【視覺微調】Logo 分區標題欄由固定寬度改為依文字內容自動收合，減少「主辦單位」「共同主辦單位」「贊助單位」與第一個 Logo 之間不必要的空白。

【扁平化輸出】略微降低 Logo row 的垂直高度與上下 padding，讓網站輸出更接近扁平資訊頁，而不是單純堆疊素材。

【版面保留】維持 v9.2 的電腦版五欄、手機版四欄，以及 Logo 彼此間距設定，避免大量廠商 Logo 佔用過多版面。

【文件更新】更新 README 與產生器內 Logo 操作說明，標記目前 production version 為 v9.3。

[v9.2] 2026-06-01 - Logo 密集排版與瀏覽負擔優化

【CP Team 規格調整】Logo 區改為密集 grid 排版：依照 CP Team 針對大量廠商 Logo 的上稿情境，將輸出後的電腦版 Logo 改為一排五個，減少贊助商較多時佔用的垂直版面。

【手機閱讀優化】手機版 Logo 改為一排四個，讓網站訪客不需要滑過過長的 Logo 區，降低閱讀負擔並保留贊助商曝光。

【視覺間距優化】縮短 Logo 分區文字與第一個 Logo 之間的距離，讓主辦、共同主辦、贊助區塊更緊湊。

【聯名 Logo 支援】聯名 Logo 在輸出時改為佔用兩格寬，兼顧密集排版與長型 Logo 可讀性。

【文件更新】更新 README 與產生器內 Logo 操作說明，標記目前 production version 為 v9.2。

[v9.1] 2026-06-01 - Logo 輸出修復與交接穩定版

【Bug 修復】Logo 輸出資料來源修正：修復 Logo 在產生器內可預覽、但輸出 HTML 未包含圖片的問題。輸出現在會正確使用標準化後的 trimmedSrc 或原始 origSrc。

【架構整理】Logo 系統去重：清理重複定義的 Logo 處理函式，避免後定義函式覆蓋前定義函式造成維護誤判。

【CSS 修復】輸出樣式 selector 修正：將 `.st26*` 修正為 `.st26,.st26 *`，確保輸出 HTML 的 scoped reset 正常套用。

【草稿修復】URL Logo 狀態保存：載入草稿時會保留 URL Logo 狀態，避免 URL 欄位還原不完整。

【安全性】輸出連結防護：新增 URL 基本檢查，阻擋 javascript: 等不安全連結，並為外部連結加入 rel="noopener noreferrer"。

[v8.3] 2026-05-12 - 預覽模組修復與最終定案

【Bug 修復】TypeError 排除：修復 generate 函式中因尋找舊版 DOM 容器導致的 Cannot read properties of null 錯誤，確保輸出與預覽流程切換順暢。

【介面還原】雙裝置預覽歸位：完整接回「手機端 / 電腦端」即時預覽切換按鈕，與對應之 RWD CSS 框架。

[v8.2] 2026-05-11 - 視覺降維與動態前綴整合

【架構優化】地點與錨點膠囊化融合：將原先厚重的「地點 (Venue)」紫底色塊移除，降維成白底紫框空心膠囊。將其與「票價與報名」、「講師陣容」、「詳細議程」收束於同一水平導覽列中，徹底消除視覺千層糕效應。

【動態前綴】無污染主題標籤：利用 CSS ::before 偽元素，在不更動底層 DOM 數據的前提下，於前台自動生成「主題：」或「Theme: 」前綴，強化資訊識別度。

【安全防護】跳脫字元重構：修復 JavaScript 模板字面量拼接時造成的 SyntaxError 斷點，防堵 HTML 解析器異常中斷。

[v8.1] 2026-05-11 - 語法報錯排除

【Bug 修復】參數異常排除：修正 renderLogo 函式中多餘字元導致的 missing ) after argument list 語法錯誤。

[v8.0] 2026-05-11 - CMS 視覺適配重構 (v6.6 UI 歸位)

【介面還原】：將前端 UI 徹底退回視覺最完善的 v6.6 狀態，包含場地雙語預覽、票價表格預覽、以及拖放式 Logo 上傳模組。

【資料流鎖定】：底層維持 v7.9 的 0511(L) 絕對座標安全解析引擎，兼顧後端資料擷取穩固與前端介面完善。
