ST26 論壇 HTML 產生器更新日誌

[v9.9] 2026-06-04 - 合作單位名稱與連結容錯修正

【協辦單位修正】修復 AI 將「協辦單位」誤填為單位名稱時，輸出只剩紫色「協辦單位」文字的問題；角色標籤不會再被當成單位名稱發布。

【連結修正】合作單位 URL 支援未加 https:// 的網址，輸出時會自動補成安全可點擊連結。

【名稱回推】針對目前合作論壇常見單位網址新增名稱回推，若 AI 吃掉 NTU 協辦單位名稱但保留網址，輸出會回補正確單位名稱。

【AI 規則】補強 AI 解析規則，明確禁止輸出「Co-Organizer | Co-Organizer | URL」這類角色重複格式。

[v9.8] 2026-06-04 - 輸入入口減法與合作論壇跑版修正

【輸入減法】Step 1 入口由三個簡化為兩個：「有 Excel 資料」與「沒有 Excel / Word 內容」，降低上架人員一開始的判斷負擔。

【跑版修正】合作單位支援延續上一個角色；若第二個協辦單位只貼「單位名稱」或「單位名稱 ｜ URL」，系統會自動沿用上一列的協辦單位角色，不再把單位名稱誤放到左側標籤欄。

【連結回歸文字】撤回晚宴報名與相關網站的獨立按鈕輸出，改回跟著備註或大綱文字自然顯示並自動連結，避免加入對方未要求的額外版面。

【AI 修正】AI 解析規則調整為將晚宴報名、協會官網等外部連結保留於備註文字，不再抽成獨立 related links；若仍收到 related links，會自動併回備註欄。

[v9.7] 2026-06-04 - 合作單位角色分組與連結輸出

【合作單位分組】合作論壇輸出中，相同角色會自動合併為同一區塊；多個「協辦單位 / Co-Organizer」不再重複顯示左側角色標籤。

【單位連結】合作單位欄位維持「角色 ｜ 單位名稱 ｜ URL」格式，每個單位若提供 URL，輸出時會各自成為可點擊超連結。

【操作提示】更新合作單位欄位說明與 placeholder，示範多個協辦單位可用相同角色分行填寫，輸出會自動整理。

[v9.6] 2026-06-04 - 一般論壇與合作論壇分流編輯

【分流編輯】Step 2 依論壇類型自動切換欄位，一般論壇與合作論壇不再把所有選填項目混在同一個表單中。

【一般論壇減法】一般論壇流程收起合作單位文字連結與相關連結欄位，避免 CP Team 處理 Excel 場次時被合作論壇欄位干擾。

【合作論壇減法】合作論壇流程收起主席/主持人與一般論壇輸出細項，保留基本資訊、主題大綱、合作單位、Logo、報名與外部連結等上架必要欄位。

【交接原則】README 新增工作流程分流說明，明確標示產生器應定義乾淨資料與編輯流程，而不是讓上架人員吸收所有來源格式混亂。

[v9.5] 2026-06-04 - 合作論壇入口獨立化

【入口調整】在 Step 1 新增「合作論壇 / Word 內容」輸入方式，避免合作論壇功能藏在一般 AI 文字輸入或 Step 2 設定中。

【操作流程】選擇合作論壇入口後，AI 解析會自動進入合作論壇模式，主題與大綱直接展開，並關閉不適用的 SEMI 自辦票價附加資訊。

【AI 欄位擴充】AI 解析新增合作單位文字連結與相關連結欄位，方便從 Word/email 來源抽出主辦、協辦、晚宴報名、協會官網等外部連結。

[v9.4] 2026-06-04 - 合作論壇模式與超連結輸出

【合作論壇模式】新增合作論壇模式，讓外部或共同主辦論壇可保留完整主題、大綱與說明內容，但仍透過統一版型輸出。

【大綱直接展開】合作論壇模式下，主題與大綱改為直接顯示，不再預設收合於 accordion，適合內容完整、需直接對外說明的合作論壇頁面。

【超連結保留】大綱與報名備註支援自動連結 URL、email 與 Markdown 連結格式，避免 Word 來源中的重要外部連結在輸出時變成不可點擊文字。

【合作單位文字連結】新增合作單位文字連結欄位，Logo 尚未收到時可先發布主辦/協辦單位名稱與連結，後續再補 Logo。

【相關連結】新增中英文相關連結欄位，可放置晚宴報名、協會官網、外部活動頁等合作論壇常見連結。

【票價輸出】合作論壇模式會自動關閉 SEMI 自辦論壇的 Day Pass 與團報優惠文案，避免外部論壇套用不適用資訊。

[v9.3] 2026-06-02 - Logo 標題間距與扁平化輸出微調

【視覺微調】Logo 分區標題欄由固定寬度改為依文字內容自動收合，減少「主辦單位」「共同主辦單位」「贊助單位」與第一個 Logo 之間不必要的空白。

【扁平化輸出】略微降低 Logo row 的垂直高度與上下 padding，讓網站輸出更接近扁平資訊頁，而不是單純堆疊素材。

【版面保留】維持 v9.2 的電腦版五欄、手機版四欄，以及 Logo 彼此間距設定，避免大量廠商 Logo 佔用過多版面。

【文件更新】更新 README 與產生器內 Logo 操作說明，標記目前 production version 為 v9.3。

[v9.2] 2026-06-01 - Logo 密集排版與瀏覽負擔優化

【CP Team 規格調整】Logo 區改為密集 grid 排版：依照 CP Team 針對大量廠商 Logo 的上稿情境，將輸出後的電腦版 Logo 改為一排五個，減少贊助商較多時佔用的垂直版面。

【手機閱讀優化】手機版 Logo 改為一排四個，讓網站訪客不需要滑過過長的 Logo 區，降低閱讀負擔並保留贊助商曝光。

【視覺間距優化】縮短 Logo 分區文字與第一個 Logo 之間的距離，讓主辦、共同主辦、贊助區塊更緊湊。

【聯名 Logo 支援】聯名 Logo 在輸出時改為佔用兩格寬，兼顧密集排版與長型 Logo 可讀性。

【文件更新】更新 README 與產生器內 Logo 操作說明，標記目前 production version 為 v9.2。

[v9.1] 2026-06-01 - Logo 輸出修復與交接穩定版

【Bug 修復】Logo 輸出資料來源修正：修復 Logo 在產生器內可預覽、但輸出 HTML 未包含圖片的問題。輸出現在會正確使用標準化後的 trimmedSrc 或原始 origSrc。

【架構整理】Logo 系統去重：清理重複定義的 Logo 處理函式，避免後定義函式覆蓋前定義函式造成維護誤判。

【草稿修復】URL Logo 狀態保存：載入草稿時會保留 URL Logo 狀態，避免 URL 欄位還原不完整。

【安全性】輸出連結防護：新增 URL 基本檢查，阻擋 javascript: 等不安全連結，並為外部連結加入 rel="noopener noreferrer"。
